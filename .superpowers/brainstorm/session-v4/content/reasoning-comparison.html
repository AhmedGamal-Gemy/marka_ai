Reasoning Implementation: Visual Comparison

How each approach looks in your logs and code

# Log Output:
> User: "I want to sell my new watch"
> LLM:
1. User mentions 'sell' and 'new watch'.
2. This is a request for content creation.
3. Setting intent to GENERATE_POST.
Intent: GENERATE_POST

A) Chain-of-Thought

Natural text flow. Great for debugging, but requires string parsing in Python.

// Structured JSON:
{
  "reasoning": "User expressed a clear desire to create a listing for a watch.",
  "intent": "GENERATE_POST"
}

B) Pydantic Schema

Structured JSON. Native Pydantic validation. Zero parsing errors. (Recommended)

// API Metadata:
thoughts_token_count: 85

// Final Text:
GENERATE_POST

C) Native Thinking

Cleanest output. Reasoning is "hidden" in metadata. Hardest to debug visually.

Why B is best for Marka AI

  • Type-safe (matches your Enums)
  • Reliable tool routing
  • Easy to log to MongoDB
